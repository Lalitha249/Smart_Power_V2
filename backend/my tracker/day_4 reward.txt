# 🧠 Reward Logic (Business Rules)

### Tier System

| Usage % | Reward Points |
| ------- | ------------- |
| ≤ 50%   | 100           |
| ≤ 75%   | 70            |
| ≤ 90%   | 40            |
| ≤ 100%  | 10            |
| > 100%  | 0             |

Formula:

```
usage_percent = (total_units / included_units) * 100
```

Edge Case:

* If plan_limit = 0 → reward = 0

---

# 🔄 Reward Flow Integration

Reward is automatically calculated when:

```
GET /usage/summary/<user_id>
```

Flow:

```
Add usage
     ↓
Call usage summary
     ↓
update_user_rewards()
     ↓
Reward stored in DB
     ↓
Reward available via REST endpoint
```

---

# 🌐 REST Endpoints Delivered

### 1️⃣ Get Reward

```
GET /users/<user_id>/rewards
```

Response:

```json
{
  "user_id": "user4",
  "reward_points": 70
}
```

---

### 2️⃣ Claim Reward

```
PATCH /users/<user_id>/rewards
```

Body:

```json
{
  "action": "claim"
}
```

Response:

```json
{
  "user_id": "user4",
  "claimed_points": 70
}
```

After claim → reward resets to 0.
