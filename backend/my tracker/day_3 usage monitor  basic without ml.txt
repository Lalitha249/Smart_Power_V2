🏗 1️⃣ Architecture Followed

We followed Layered Architecture:

Routes → Controller → Service → Model → MongoDB


Each layer has one responsibility:

Routes → URL mapping

Controller → HTTP handling

Service → Business logic

Model → Database queries only

DB → Data storage

No layer mixes responsibilities.

📦 2️⃣ Database Design
Collection: usage_logs

Each usage entry stored as:

{
  "user_id": "user1",
  "device_name": "AC",
  "units_consumed": 10,
  "timestamp": datetime,
  "month": "2026-02",
  "calculated_cost": 0
}

Purpose:

Store every usage event

Enable monthly aggregation

Enable device-wise grouping

Enable billing calculation

Enable future ML

We store logs individually — never overwrite totals.

📁 3️⃣ Model Layer (Database Only)
File: models/usage_model.py

Functions implemented:

✔ insert_usage_log(data)

Inserts one usage document.

✔ get_total_usage(user_id, month)

Uses MongoDB aggregation:

"$sum": "$units_consumed"


Returns total monthly units.

✔ get_device_usage_by_month(user_id, month)

Uses MongoDB pipeline:

"$group": {
    "_id": "$device_name",
    "total_units": {"$sum": "$units_consumed"}
}


Then sorts descending.

Purpose:
Identify highest energy-consuming device.

🔥 Incremental Billing Logic

We do NOT recalculate full month cost.

We calculate only cost caused by THIS entry.

Formula:
remaining_free_units = max(included_units - current_total, 0)


If:

units > remaining_free_units


Then:

extra_units = units - remaining_free_units
extra_cost = extra_units × extra_unit_cost


We store only this entry’s extra cost.

🔹 B) get_usage_summary_service(user_id)

Steps:

Validate user

Validate subscription

Fetch plan

Aggregate monthly total

Compare with included_units

Calculate:

extra_units = total_units - included_units
total_extra_cost = extra_units × extra_unit_cost


Returns full monthly summary.

🔹 C) get_device_analytics_service(user_id)

Highest usage device
